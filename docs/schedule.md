## 1. 概述

### 1.1 文档目的

本文档定义数莲 PaaS IM 系统建设项目的任务分解（WBS）、依赖关系、人力估算及里程碑规划，作为项目管理与进度跟踪的依据。

### 1.2 项目范围回顾

| 模块 | 客户端 | 服务端 | 管理后台 |
|------|--------|--------|---------|
| 基础系统能力 | App (iOS/Android) | user-svc | - |
| 会话与聊天 | App + Web | message-svc, conversation-svc | - |
| 群组管理 | App + Web | group-svc | - |
| 通讯录 | App + Web | contact-svc | - |
| 机器人 | - | bot-svc | Web 管理端 |
| 风控管理 | - | rc-svc | Web 管理端 |
| 审计日志 | - | audit-svc | Web 管理端 |
| 文件服务 | App + Web | file-svc | - |
| 卡片消息 | App + Web | card-svc | - |

### 1.3 假设条件

| 序号 | 假设 | 影响 |
|------|------|------|
| 1 | 服务端 Go 开发团队 4 人（含 1 名 Tech Lead） | 影响并行任务数 |
| 2 | iOS / Android / Web 前端各 2 人 | 三端并行开发 |
| 3 | QA 团队 2 人全程介入 | 测试不单独排期，与开发并行 |
| 4 | OpenIM SDK 集成已有基础文档 | 减少调研时间 |
| 5  | 设计师已产出全部 MasterGo 设计稿 | UI 开发可随时开始 |
