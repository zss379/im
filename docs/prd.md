# 1. 引言与全局定义
## 1.1 文档目的
本文档用于明确数莲 PaaS 平台 App升级以及IM 能力建设项目的功能范围、系统行为、交互规则及约束条件，作为产品设计、UI 设计、研发实现及测试验收的依据。

## 1.2 适用范围
本需求适用于数莲 PaaS 平台有内部沟通与协作需求的企业用户。产品功能覆盖 Web 端、iOS 端及 Android 端，适用于中小型企业。

## 1.3 术语定义
|术语|释义|
|---|---|
|IM|即时通讯|
|群组|三人以上的聊天组，分为普通群、部门群、全员群|
|SSE|Server-Sent Events，服务器推送技术|
|会话|两个或多个用户之间的通信容器|
|结构化消息卡片|一种使用预定义 JSON 格式描述，在聊天界面中以统一视觉组件展示业务信息并支持交互的消息类型。|
|Webhook|一种基于 HTTP 的回调机制，允许 IM 系统在特定事件（如消息发送、群组变更、机器人触发）发生时，向外部系统预配置的 URL 发送实时通知。外部系统也可通过 Webhook 接收消息并返回响应，实现与 IM 系统的双向集成。Webhook 常用于机器人自动回复、消息存档、业务系统联动等场景。|

# 2. 项目总体描述
## 2.1 项目概述
### 2.1.1 项目背景
当前数莲 PaaS 平台 App 端与 Web 端在消息、搜索及协作能力上存在一定差异，且 App 端部分基础系统能力尚未完善。同时，平台尚缺乏统一的即时通讯（IM）能力，无法满足用户在业务场景下的高频沟通、流程协同与消息触达需求。

随着 AI 能力在业务系统中的持续融合，平台也需要构建以 AI 互动为重要入口的智能交互能力，为用户提供更加高效、便捷的信息获取与业务处理体验，从而进一步提升平台整体协同效率与智能化水平。

因此，有必要在现有 App 1.5 版本基础上，对移动端与 Web 端进行能力补齐与功能扩展，同时建设统一的 IM 通讯体系，以实现多端能力一致、沟通协同顺畅，并持续提升平台整体的使用体验与协作效率。

### 2.1.2 项目目标
本项目旨在构建统一的跨端即时通讯能力，并完善基础系统功能，具体目标如下：
1. **能力一致**：补齐 App 端与 Web 端之间的功能差异，实现多端能力一致；
2. **构建IM**：建设平台级 IM 通讯能力，支持单聊、群聊及多类型消息交互；
3. **安全可控**：提供 IM 管理与风控能力，满足平台运营与安全管控需求；
4. **体验升级**：完善 App 端基础系统能力，提升整体使用体验与稳定性。

## 2.3 运行环境
### 客户端
- iOS：iOS 12.0 及以上（OpenIM iOS SDK最低版本）
- Android：Android 5.0 (API 21) 及以上（OpenIM Android SDK最低版本）

### 服务端
#### 操作系统
Linux：CentOS 7 / Ubuntu 18.04 / Ubuntu 20.04
Windows：Windows7+

#### 运行依赖
- Go 1.18+
- Docker 19.03+（容器部署）

#### 数据库
- MySQL 8.0
- Redis 7.0
- MongoDB 7.0

#### 中间件
- Kafka 3.5.1（消息队列）
- Nginx（反向代理、负载均衡）

### 网络要求
1. 支持IPv4/IPv6双栈；
2. 弱网（2G/3G）保障文本消息自动重试，遵循OpenIM弱网策略；
3. 客户端与服务端基于WebSocket长连接，自动重连、消息可靠投递。

# 3. 系统总体架构
## 3.1 架构概述
三层分层架构：
1. **客户端**：App / Web，负责UI交互、消息渲染；
2. **IM服务端**：消息路由、持久化、离线推送、多端同步；
3. **业务系统**：通过API/Webhook与IM双向集成。

## 3.2 技术通信架构
- 客户端实时通信：WebSocket
- 离线消息拉取：HTTP
- 机器人事件回调：Webhook

## 3.3 数据同步原则
1. 全量消息App/Web多端实时同步；
2. 会话未读数跨端统一；
3. 用户在线/自定义状态实时同步。

# 4. 功能需求
整体分为五大模块：基础系统能力、会话与聊天、群组管理、通讯录、机器人&风控后台；所有会话、消息数据全端同步，群组严格角色权限管控，后台支持审计追溯。

## 4.4 基础系统能力
### 4.4.1 概述
基于App1.5版本补齐缺失基础功能，统一多端差异，为IM、业务卡片提供底层支撑：启动页、登录、全局搜索、缓存、用户状态、租户切换、推送复用、离职用户处理。

### 4.4.2 分项功能说明
#### 1. 静态启动页
1. 流程：点击App图标 → 展示1s品牌启动页 → 已登录跳首页/未登录跳登录；
2. 规则：无交互操作，不阻塞后台初始化；
3. 异常：启动资源加载失败不影响程序启动。

#### 2. 登录页
能力：账号密码登录、记住密码、隐私政策勾选、Token自动续期；
规则：未填写账号密码登录按钮置灰；Token失效强制跳转登录；错误提示明确（账号错误/网络异常）。

#### 3. 全局统一搜索
**检索范围**：聊天记录、联系人、群组、业务菜单、流程；
**展示规则**
1. 结果固定分组顺序，每组最多3条，超出显示「搜索更多」；
2. 关键词高亮标黄；
3. 点击对应条目直接跳转对应会话/业务页面；
4. 无匹配展示空页面提示；
**排序逻辑**
- 聊天记录：时间倒序；
- 联系人：常用优先；
- 群组：活跃度优先；
支持拼音模糊检索（zs匹配张三）。

#### 4. App首页快捷操作
会话列表右上角「+」面板：扫一扫、创建群聊，固定顺序，预留扩展位。

#### 5. 缓存分级清理
1. 全部缓存：清理图片、文件、H5本地存储；
2. 文件缓存：仅清理下载媒体；
3. H5缓存：清理内嵌网页Cookie、LocalStorage；
操作前弹出二次确认弹窗。

#### 6. 用户在线/自定义状态
##### 状态分类
1. **系统自动状态**
   - 在线：App/Web活跃；
   - 离线：Web30分钟无交互、未登录App；
2. **用户手动状态**：忙碌、请勿打扰；
##### 优先级（从高到低）
离线 > 请勿打扰 > 忙碌 > 在线；
##### 展示位置
单聊顶部、个人名片、会话列表头像标识；
##### 同步规则
切换后全端实时同步；个人状态弹窗一键修改。

#### 7. 左上角头像租户/服务器切换
1. 弹窗展示：头像（无头像取末尾文字）、姓名、部门、当前状态；
2. 操作入口：修改状态、切换租户、切换服务器；
3. 规则：
   - 切换后清空本地会话缓存、断开IM连接，重新登录；
   - 多租户才显示切换按钮；服务器切换仅内测/调试包可见；
   - 切换前弹窗确认退出当前账号。

#### 8. 用户离职/账号删除处理规则
|场景维度|已离职用户|账号删除用户|
|---|---|---|
|通讯录展示|头像置灰，不可发起聊天|完全隐藏|
|单聊会话|保留会话，提示账号停用，输入框锁定|会话保留，提示用户不存在|
|群聊处理|自动退群，历史消息标注「（已离职）」|自动退群，统一显示「已删除用户」|
|群主权限|自动转让给最早管理员|自动转让给最早管理员|
|用户自建机器人|全部停用，群内通知|全部删除，群内通知|
|文件策略|单聊未下载文件清理；群文件保留|单聊文件清理，群文件上传者匿名|
|审计日志|完整保留姓名ID，标注离职|仅保留ID，姓名替换匿名|

#### 9. 推送复用极光JPush
##### 复用规则
现有JPush注册、设备绑定逻辑完全复用，不新增推送SDK；
##### 推送分级策略
1. **前台App内**：非当前会话弹出顶部横幅，当前会话不推送；
2. **后台/离线**：下发系统通知；
3. **免打扰规则**：普通消息抑制推送，@我/@所有人、审批、系统通知强制穿透；
##### 推送摘要规则
- 文本：截取前50字符；
- 图片/视频/语音：统一标记类型；
- 卡片消息：展示卡片标题；
- 撤回消息不推送；
##### 离线聚合
同一会话多条消息合并推送，离线汇总展示未读总数；
##### 点击跳转逻辑
推送点击直接进入对应会话，会话已销毁则跳转会话首页；账号停用推送强制跳转登录页。

## 4.5 会话与聊天模块
### 4.5.1 会话列表页面布局
**顶部**：左上角头像弹窗入口、顶部搜索框、右上角创建群聊+扫一扫；
**主体**：会话列表，置顶会话置顶展示，按最新消息时间倒序；
**底部导航**：首页、通讯录、工作台、我的；
单条会话展示项：头像、会话名、最新消息摘要、时间、未读数字、免打扰图标。

### 4.5.2 会话操作能力
左滑操作：置顶/取消置顶、标已读/标未读、删除；
会话设置内：开启/关闭免打扰；
约束：删除仅清除本地列表，云端消息永久存储，全局搜索仍可检索。

### 4.5.3 全类型消息支持
1. **基础消息**：文本、Emoji、图片、视频、文件、语音、名片、系统提示；
2. **高级消息**：结构化业务卡片、SSE流式AI文本；
### 4.5.4 消息发送状态
发送中：加载转圈；发送成功无标识；失败红色感叹号，支持手动重试；
弱网自动重试3次，失败留存本地待发队列。

### 4.5.5 消息操作
撤回（2分钟时效）、复制文本、单条/多选合并转发、引用回复、双击放大媒体；
单聊支持已读回执，群聊不展示已读人数。

### 4.5.6 聊天输入交互
1. 输入框自适应，超过3行可全屏编辑；
2. 支持表情、拍照、本地文件、语音录制；
3. 群内输入@唤起成员列表，@所有人仅限管理员；
4. 文本上限5000字符，单次图片最多9张，单图上限10MB。

### 4.5.7 单聊专属管理
会话右上角菜单：设置备注、置顶、免打扰、查找聊天记录、快速建群、分享对方名片。

### 4.5.8 多端同步规则
1. WebSocket实时同步所有消息；
2. 任意端标记已读，全端未读数同步清零；
3. 多设备同时登录均可收发消息，状态统一。

## 4.6 群聊管理模块
### 4.6.1 群创建两种入口
1. 首页「+」→ 创建群聊；
2. 单聊右上角菜单 → 创建群聊；
创建至少选择2人，创建者自动为群主，默认群名为成员名称拼接。

### 4.6.2 群角色三级权限
1. **群主**：全部管理权限（增删、改名、转让、解散、设置管理员）；
2. **管理员**：增删普通成员、发布公告、成员禁言；不可移除群主；
3. **普通成员**：基础聊天；添加成员、@所有人、修改群名需管理员开放权限。

### 4.6.3 群核心配置项
1. 群名称（30字符上限）、群头像、群描述；
2. 群二维码：保存、分享、刷新重置；
3. 群公告：富文本，置顶，支持历史记录；
4. 入群验证：无需审批/管理员审批；
5. 权限开关：控制全员能否改名、@所有人、添加成员；
6. 群主转让、管理员数量管控（上限成员10%）；
7. 退出群聊（群主必须转让才可退出）、解散群聊（仅群主）。

### 4.6.4 入群申请完整生命周期
状态：pending待审核 → approved通过 / rejected拒绝 / expired过期；
1. 申请有效期默认7天（1~30天可配置）；
2. 单用户24小时最多3次有效申请（rejected计入，expired不计）；
3. 待处理申请上限50条；
4. 管理员统一后台列表处理，推送提醒；
5. 记录保留：approved永久、rejected/expired保留90天。

### 4.6.5 群辅助功能
1. 单群内消息检索：关键词+类型筛选，定位消息高亮；
2. 群免打扰：关闭推送，@消息默认穿透；
3. 折叠群会话：移入折叠分组，自动开启免打扰，置顶会话不可折叠；
4. 群机器人管理：添加/删除Webhook自定义机器人，单群最多5个。

2. 流程助手：审批、流程超时提醒；
3. 文件助手：文件上传下载通知；
管理员后台可启停，消息受风控限制。

### 4.9.2 用户自定义机器人
1. 租户最多创建20个；
2. 配置项：名称、头像、Webhook地址、API密钥、访问权限；
3. 支持启用/停用、重置密钥、删除；
4. 可添加至群聊，支持@触发。

### 4.9.3 群Webhook机器人
群设置内生成独立回调URL，外部系统POST推送消息；单群最多5个机器人，超时3次重试，重置Token旧地址失效。

### 4.9.4 聊天记录审计查询（管理员）
多条件组合检索：发送人、接收群/用户、时间区间、消息关键词、消息类型、是否命中敏感词；支持结果导出，用于合规追溯。

### 4.9.5 风控统一配置
1. **文件大小限制**：图片10MB、视频100MB、普通文件50MB，区间可配置；
2. **消息频控**：用户1s最多5条，机器人1s最多10条，超限返回429；
3. **敏感词管理**：批量导入导出，三种策略（替换***、拦截、仅记录日志），全消息前置校验。
