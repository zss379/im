## 1. 概述

### 1.1 文档目的

本文档定义数莲 PaaS 平台 IM 系统的测试策略与用例设计，覆盖功能测试、集成测试、性能测试、安全测试、兼容性测试五大维度，作为 QA 团队编写具体测试用例及执行验收的纲领性依据。

### 1.2 测试范围

| 模块 | 测试范围 | 优先级 |
|------|---------|-------|
| 基础系统能力 | 启动页、登录、全局搜索、缓存清理、个人状态、租户切换 | P0 |
| 会话与聊天 | 会话列表、消息收发、消息操作、多端同步 | P0 |
| 群组管理 | 群创建/解散、成员管理、角色权限、入群申请 | P0 |
| 通讯录 | 组织架构、成员搜索、名片 | P1 |
| 机器人管理 | 系统机器人、自定义机器人、Webhook | P1 |
| 风控管理 | 敏感词、频控、文件限制 | P0 |
| 审计日志 | 管理员日志、消息审计 | P1 |

### 1.3 测试策略

| 测试类型 | 执行方式 | 覆盖目标 |
|---------|---------|---------|
| 单元测试 | 自动化 (Go test / Jest) | 服务端核心逻辑、敏感词引擎、频控算法 |
| 接口测试 | 自动化 (Postman / Go Test) | 全部 REST API 端点 |
| 功能测试 | 手动 + 半自动化 | 端到端业务流程 |
| 多端同步测试 | 手动 | iOS/Android/Web 三端同步一致性 |
| 性能测试 | 自动化 (k6 / JMeter) | 消息 TPS、延迟、并发 |
| 安全测试 | 手动 + 工具扫描 | 渗透测试、权限越权、SQL 注入 |
| 兼容性测试 | 手动 | 多设备/多系统/多浏览器 |

## 7. 缺陷分级标准

| 级别 | 定义 | 响应时间 | 修复时限 |
|------|------|---------|---------|
| **P0 致命** | 核心功能不可用，如无法登录、消息丢失、全端崩溃 | 立即响应 | 4h |

| **P3 轻微** | UI/文案问题、体验不佳但功能正常 | 24h | 下一迭代 |
